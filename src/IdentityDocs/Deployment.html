Prerequisites

Comsoft Identity server prerequisites:

  • Windows Server 2012 or 2016
  • Internet Information server (IIS)
  • SQL Server 2012 or later
  • Microsoft SQL Server Management Studio
  • .NET Core 2.1 hosting runtime
  • SSL Certificate

Fresh Deployment

Fresh Deployment

This section will describe how to configure and install the application over fresh server only , for regular deployment please refer to section Regular deployment

SSL Certificate

In order to import the certificate you need to access it from the Microsoft Management Console (MMC), then you need to follow the below steps:

  • Open the MMC (Start > Run > MMC).
  • Go to File > Add / Remove Snap In
  • Double Click Certificates
  • Select Computer Account.
  • Select Local Computer > Finish
  • Click OK to exit the Snap-In window.
  • [+] next to Certificates > Personal > Certificates
  • Right click on Certificates and select All Tasks > Import
  • Click Next
  • Click Browse
  • Select the .cer, .crt, or .pfx you would like to import. Click Open.
  • Click Next
  • Select Automatically select the certificate store based on the type of certificate.
  • Click Finish & OK

The certificate is now visible in IIS.

Internet Infromation Service (IIS) features

  • Open Server Manager
  • You can search for it under the Start menu
  • In Server Manager, under Quick Start, click “Add roles and features”. The “Add Roles and Features Wizard” window pops up.
  • Click Next.
  • Under Select installation type, check “Role-based or feature-based installation”.
  • Under Select destination server, select the server which you want to enable IIS and click Next.
  • Under Server Roles, check “Web Server (IIS)”.

Internet Infromation Service (IIS) Site

The next step is to create IIS site to host our application:

  • In IIS, under “Sites”, right click and select “Add Website” to open the Add Website dialog.
  • Under Site name, enter “ComsoftIdentityServer”.
  • Click “Select” to open the “Select Application Pool” dialog and select the “ComsoftIdentityServer” pool and make sure that the .NET CLR version option value is "No Managed Code".
  • Under physical path, click “Browse” and select the path to the published app "C:\Identity Server\IIS files\Identity server Site"
  • Enter the information for the binding. For more details, see the below screenshot.

Published files

Copy the published files to the location "C:\Identity Server\IIS files\Identity server Site" on the server and make sure that IIS user granted permission to read/write on the root folder.

Configuration files

Open appsettings file and edit the below settings:

  • Connection string
  • Certificate Name: the certificate name represents the thumbprint; you can get the thumbprint value from IIS/root site/server certificate then right click on the targeted certificate and scroll down to thumbprint property then copy the value without spaces, please check the below screenshot.

Regular deployment

The regular deployment will be very simple, all you need is to copy the published files to “C:\Identity Server\IIS files\Identity server Site” folder except appsettings.json and web.config files .

Backup previous build files

Make sure to backup all previous build files before replacing the new published files
